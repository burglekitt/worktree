# Troubleshooting

## Build fails with `next-mdx-import-source-file`

If you see this when running docs locally, verify `docs/next.config.mjs` includes the Turbopack resolve alias:

```js
resolveAlias: {
  "next-mdx-import-source-file": "./mdx-components.tsx",
}
```

## GitHub Pages serves broken links

Confirm production base path is set to your repo name (for this repo: `/worktree`) and local dev uses an empty base path.

## Search does not return results

Search uses Pagefind generated during `docs:build`. Ensure postbuild runs and `_pagefind` exists in the exported output.

## Formatting on save does not work for MDX

Check that:

1. The MDX extension is installed.
2. `mdx` is mapped to Prettier in workspace settings.
3. Format on save is enabled for the workspace.

## Lockfile confusion in workspace

Use the root `pnpm-lock.yaml` as the canonical lockfile and avoid committing a separate lockfile under `docs/`.
